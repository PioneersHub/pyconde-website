# Content Strategy — One Authoritative Domain

> Hand-off brief. Read this before touching talks, speakers, editions, URLs, or
> redirects. It explains *why* the site is structured the way it is, so your
> changes reinforce the strategy instead of fighting it.

## TL;DR

We are consolidating **every edition's talks and speakers onto one living domain**
(the current-edition site, `pycon.de` / today `2026.pycon.de`) instead of leaving
them stranded on frozen per-year sites. The per-year `YYYY.pycon.de` sites are
demoted to **archives that redirect**: their session and speaker pages `301` to
the canonical page on the main site. One domain now holds the full history
(2016→present), keeps growing every edition, and accumulates all the search
authority, AI-citation surface, and internal-link equity that used to be split
across a dozen dead subdomains.

## The shift

**Before — fragmented authority.** Each edition lived on its own domain
(`2017.pycon.de`, `2018.de.pycon.org`, `2026.pycon.de`, …). When an edition
ended, its site froze forever. Ten years of talks and speakers = ten isolated,
never-updated domains, each holding a thin slice of content, none reinforcing
the others. Search engines and AI answer-engines saw a graveyard of small,
static sites; link equity and crawl budget were scattered and decaying.

**After — compounding authority.** One canonical domain hosts the complete
corpus. Every edition is imported under `/archive/{year}/` with its talks,
speakers, tracks, transcripts, and recordings — cross-linked and full-text
searchable. Each new edition *adds depth to the same domain* rather than
starting a new island. The old per-year domains keep their inbound links alive
by redirecting into this corpus, so their historical authority flows *into* the
canonical home instead of rotting.

This is the moat: ~800 talks, ~900 speaker pages, ~550 speaker-diarized
transcripts, linked recordings, per-track hubs, and full-text search — all under
one roof, all reinforcing each other's ranking. No competing conference has
this; they still go dark for ten months a year across dead subdomains.

## Canonical model — where content lives

The main site is the **single source of truth**. Canonical host:
`https://2026.pycon.de` (`databags/branding.yaml → canonical_host`; rolls forward
with each edition — the *current* edition's domain is always the canonical home).

```
/                                   current edition (marketing + programme)
/talks/{slug}/                      current-edition talks
/speakers/{code}/                   current-edition speakers
/archive/                           index of all past editions
/archive/{year}/                    one past edition (Event schema, EventCompleted)
/archive/{year}/talks/{slug}/       past talk   ← CANONICAL home of that talk
/archive/{year}/speakers/{code}/    past speaker
/archive/{year}/tracks/{slug}/      per-track collection page
/search/                            full-text search across every edition (Pagefind)
```

Editions currently archived: **2016, 2017, 2018, 2019, 2022, 2023, 2024, 2025**
(2020/2021 did not happen; 2026 folds into the archive after its cycle).

Every talk page carries `rel="canonical"` pointing at *itself on this domain* —
that is the authoritative URL. Nothing on an old `YYYY.pycon.de` domain is
canonical anymore.

## Redirect model — old world → canonical home

Three redirect layers move visitors and link equity to the canonical page:

1. **Cross-domain (the strategic one).** Old per-year sites redirect their
   session/speaker URLs to the canonical archive page. Example:
   `https://2017.pycon.de/schedule/talks/an-introduction-to-pymc3/`
   → `301` → `https://2026.pycon.de/archive/2017/talks/an-introduction-to-pymc3/`.
   These live in the *old sites'* web-server config (or a static shell), not in
   this repo, but this repo owns the target URL structure they point at.

2. **In-domain slug canonicalisation.** Talks are slug-canonical. The old
   Pretalx-code URL `301`s to the slug URL, e.g.
   `/archive/2025/talks/3VYSMS/` → `/archive/2025/talks/you-dont-think-about-…/`.
   Generated by `utils/migrate_pretalx_slugs.py` (a sibling `_model: redirect`
   folder per code).

3. **Arbitrary path redirects.** Renamed/retired pages, declared in
   `databags/redirects.yaml`, rendered by `utils/generate_redirects.py`.

Redirects render as `_model: redirect` pages (`templates/redirect.html`:
meta-refresh + JS `location.replace` + `rel="canonical"` to the target +
`robots: noindex,follow`) **and** as real server-side `301`s: the generators
emit `site-config/*.nginx.conf` and `site-config/Caddyfile.*` snippets for the
hoster to install. Redirect pages are excluded from the sitemap (they are
noindex 301 stubs).

## What already exists in this repo (don't rebuild it)

- **Archive content**: all 8 editions under `content/archive/{year}/`.
- **Import pipelines**: `utils/talks.py` + `utils/speakers.py` (Pretalx, year-keyed
  config in `databags/pretalx.yaml`); `utils/import_legacy_editions.py` (2016/2017,
  web-sourced — no Pretalx); `utils/sync_recordings.py` (YouTube playlist match);
  `utils/import_transcripts.py`; `utils/download_audio.py`; `utils/generate_tracks.py`;
  `utils/social_card_img_gen.py`.
- **Slug + redirect machinery**: `utils/migrate_pretalx_slugs.py`,
  `utils/generate_redirects.py`, `templates/redirect.html`, `models/redirect.ini`,
  `databags/redirects.yaml`.
- **Discoverability**: segmented sitemap index (`/sitemaps/*.xml`), rich JSON-LD
  (Event, Person, VideoObject, FAQPage, BreadcrumbList, ItemList,
  WebSite+SearchAction, Organization), `rel="canonical"` everywhere,
  `/llms.txt` + `/llms-full-archive.txt` (full plain-text talk+transcript corpus),
  `robots.txt` explicitly allowing GPTBot/ClaudeBot/PerplexityBot/Google-Extended/CCBot.
- **Search**: Pagefind, client-side, indexes transcript bodies (`/search/`).

## Invariants — rules to respect when working in this repo

1. **The main domain is canonical. Never make an old `YYYY.pycon.de` URL
   canonical.** New/changed talk & speaker content goes under `/archive/{year}/`
   on the main site.
2. **Talk URLs are slug-canonical.** Never expose a Pretalx-code URL as the real
   page; it must `301` to the slug. Re-run `migrate_pretalx_slugs.py` after any
   fresh import.
3. **A redirect is never canonical and never in the sitemap.** `_model: redirect`
   pages are `noindex,follow` 301 stubs.
4. **One canonical page per talk/speaker — no duplication across domains.** The
   old site redirects; it must not serve a competing indexable copy.
5. **Content is generated, not hand-edited.** Talk/speaker `contents.lr` come from
   the importers; edits belong upstream (Pretalx) or in the importer, else the
   next run overwrites them. (2016/2017 are the exception — sourced from the web
   and encoded in `import_legacy_editions.py`.)
6. **Configurable values live in YAML databags**, never as literals in templates
   or Python.

## Adding a new edition (the repeatable workflow)

1. Add the edition's Pretalx slug + question/submission-type IDs to
   `databags/pretalx.yaml` (year-keyed).
2. `uv run python utils/talks.py --year YYYY` and `utils/speakers.py --year YYYY`.
3. `uv run python utils/migrate_pretalx_slugs.py` (slug URLs + code→slug redirects).
4. `uv run python utils/generate_tracks.py`.
5. When recordings land: `utils/sync_recordings.py`; when transcribed:
   `utils/import_transcripts.py`.
6. Add the edition to `databags/past_editions.yaml`, the footer year row in
   `templates/layout.html`, and the archive-pointer year range.
7. Stand up the old `YYYY.pycon.de` cross-domain `301` redirects to the new
   `/archive/{year}/…` pages (hoster-side).
8. `make build` (regenerates tracks, redirects, sitemaps, Pagefind index).
